### 1. Ошибка `no such table: billing_history` ✅

**Статус:** ✅ **ИСПРАВЛЕНО**

> ⚠️ **Критичность:** Блокировала загрузку DBF файлов

**Ошибка:**
```
[CONTROLLER _log] ERROR: Критическая ошибка: no such table: billing_history
```

**Причина:** Метод `insert_billing()` в `core/database.py` пытался вставлять данные в несуществующую таблицу `billing_history`.

**Решение:** Метод разделён на две вставки:
```python
def insert_billing(self, records: List[Dict]):
    """Вставка записей биллинга в две таблицы: charges и address_linking."""
    for record in records:
        # Вставка в charges (начисления)
        self.cursor.execute('''
            INSERT OR IGNORE INTO charges
            (account_id, period, charge, saldo, source_file)
            VALUES (?, ?, ?, ?, ?)
        ''', (...))

        # Вставка в address_linking (справочник адресов)
        self.cursor.execute('''
            INSERT OR REPLACE INTO address_linking
            (account_id, fio_raw, surnames, address_raw, address_norm)
            VALUES (?, ?, ?, ?, ?)
        ''', (...))
    self.conn.commit()
```

**Файл:** `core/database.py` (строка 365)

**Результат:**
- ✅ Загружено DBF: 1 298 591 запись в `charges`
- ✅ Загружено адресов: 36 681 запись в `address_linking`

### 2. Ошибка `AttributeError: 'ProcessingController' object has no attribute 'get_segmentation_results'` ✅

**Статус:** ✅ **ИСПРАВЛЕНО**

**Ошибка:**
```
File "c:\Projects\DebtManagerApp2\gui\tab_segment.py", line 296, in _refresh_data_safe
    self.segmentation_df = controller.get_segmentation_results()
                           ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
AttributeError: 'ProcessingController' object has no attribute 'get_segmentation_results'
```

**Причина:** В классе `ProcessingController` отсутствовал метод для получения полного DataFrame сегментации.

**Решение:** Добавлен метод в `core/gui_controller.py`:
```python
def get_segmentation_results(self) -> Any:
    """Получение полного DataFrame с результатами сегментации."""
    # Сначала пробуем вернуть из кэша
    if self._cached_segmentation_df is not None:
        return self._cached_segmentation_df

    # Если кэш пуст, пробуем из процессора
    if self.processor:
        try:
            return self.processor.get_segmentation_results()
        except Exception as e:
            return None

    # Процессор закрыт и кэш пуст
    return None
```

**Файл:** `core/gui_controller.py` (строка 577)

**Требуется:**
- [ ] Исправить маппинг колонок в загрузчике Excel
- [ ] Проверить таблицу `raw_fssp` на наличие поля `case_number` (номер ИП)
- [ ] Обновить отображение в таблице на вкладке «Загрузка»

### 5. Вкладка "Сегментация" — Журнал событий не обновляется

| Параметр | Значение |
|:---------|:---------|
| **Ожидалось:** | Логи в реальном времени при обработке |
| **Фактически:** | Только "Ожидание запуска обработки..." |
| **Где:** | `gui/tab_segment.py`, метод `add_log()` |

**Применённое исправление:**
```python
# main.py
def _update_segment_log(self, message, level):
    seg_tab = self.frames.get("SegmentTab")
    if seg_tab and hasattr(seg_tab, "add_log"):
        seg_tab.add_log(f"[{level}] {message}")

# gui_controller.py
if self.segment_tab_ref:
    self.segment_tab_ref.add_log(f"[{level}] {message}")
```

<details>
<summary>🔍 Логи диагностики (нажми меня)</summary>

```
[MAIN] SegmentTab = .!ctkframe.!segmenttab  ← Ссылка есть
[MAIN] SegmentTab linked to controller      ← Контроллер установлен
[CONTROLLER _log] Вызов add_log...          ← Вызов идёт
[SegmentTab add_log] Получено: ...          ← Метод вызывается
[SegmentTab add_log] Успешно добавлено      ← Ошибок нет
```

</details>

**Проблема:** Логи добавляются в textbox, но пользователь их не видит.

**Возможные причины:**
- textbox остаётся в состоянии `disabled` и текст не отображается
- `see("end")` не прокручивает к новому тексту
- textbox перекрывается другим элементом

**Требуется:**
- [ ] Проверить, что textbox виден и текст фактически вставляется
- [ ] Убрать `state="disabled"` на время вставки
- [ ] Проверить работу `see("end")`

### 6. Автоматическое переключение на вкладку "Журнал ошибок"

| Параметр | Значение |
|:---------|:---------|
| **Ожидалось:** | Обработка в фоне без переключения вкладок |
| **Фактически:** | При запуске обработки переключает на LogsTab |
| **Где:** | `main.py`, метод `start_processing()`, строка ~185 |

**Проблемный код:**
```python
if confirm:
    self.controller.start_processing(config)

    # Переключаемся на вкладку журнала для просмотра прогресса
    self.show_frame('LogsTab')  ← ❌ ПРОБЛЕМА ЗДЕСЬ
```

**Почему это плохо:**
- ❌ Пользователь работает на одной вкладке, его переключает на другую
- ❌ Невозможно наблюдать за прогрессом на вкладке "Сегментация"
- ❌ Раздражающее поведение интерфейса

**Требуется:**
- [ ] Убрать `self.show_frame('LogsTab')` из `start_processing()`
- [ ] Обработка идёт в фоне, пользователь остаётся на текущей вкладке
